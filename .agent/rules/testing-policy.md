# Development Policy: Late Testing (Always On)
Description: Prioritizes feature completion and structural integrity over early testing.

## Rules
1. **No Early Tests**: Never write unit tests or integration tests during the initial development phase of a feature.
2. **Compilation Focus**: During implementation, the only required validation is that the code is **compilable and syntactically correct**.
3. **Refactoring Tolerance**: Allow the structure to be fluid during the first 80% of the project.
4. **Final Testing Phase**: Tests should only be written and executed once the core functionality is stable and the roadmap is 90% complete.
