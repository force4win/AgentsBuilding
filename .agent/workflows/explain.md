# Explain Component (Universal)
Description: Generates a mental map or summary of a specific component or file.
Activation: Manual (/explain)

## Steps
1. **Context Collection**: Analyze the active file and its direct dependencies.
2. **Abstract Summary**: Describe "what" the component does in 1-2 sentences.
3. **Data Flow**: List inputs, main transformations, and final outputs.
4. **Integration**: Explain how it fits into the broader architecture.
5. **Output**: Format as a Markdown report for the user.
