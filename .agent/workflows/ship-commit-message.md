# Ship Changes (Universal)
Description: Summarizes current session work and prepares a standardized commit message.
Activation: Manual (/ship)

## Steps
1. **Summary Analysis**: Scan all modified files in the current workspace.
2. **Commit Generation**: Generate a concise commit message following the "Conventional Commits" standard (e.g., `feat(auth): add login validation`).
3. **Pull Request Drafting**: Create a brief PR description outlining:
   - What changed.
   - Why it changed.
   - Any manual testing performed.
4. **Validation**: Ask the user to confirm before staging changes.
